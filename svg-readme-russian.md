# SVG --- полный конспект по работе с иконками, цветом и SVG‑sprite

## Что такое SVG

**SVG (Scalable Vector Graphics)** --- это векторная графика в виде
XML‑разметки.

Пример:

``` html
<svg viewBox="0 0 24 24">
  <path d="M12 2L2 22h20L12 2z" />
</svg>
```

### Чем SVG отличается от PNG/JPG

-   SVG не теряет качество при масштабировании
-   можно менять цвет через CSS
-   можно анимировать
-   можно редактировать кодом
-   идеально подходит для иконок интерфейса

------------------------------------------------------------------------

# Способы использования SVG

## 1. Inline SVG (встроенный прямо в HTML)

``` html
<svg class="icon" viewBox="0 0 24 24">
  <path d="..." />
</svg>
```

### Плюсы

-   полный контроль через CSS
-   можно анимировать части SVG
-   можно менять fill и stroke

### Минусы

-   HTML становится длиннее
-   повторяющиеся иконки дублируют код

------------------------------------------------------------------------

## 2. SVG как картинка через img

``` html
<img src="images/icon.svg" alt="Иконка">
```

### Плюсы

-   очень просто использовать

### Минусы

-   нельзя менять цвет через CSS
-   нет доступа к path внутри SVG

------------------------------------------------------------------------

## 3. SVG как background-image

``` scss
.icon {
  background-image: url("../images/icon.svg");
}
```

### Плюсы

-   удобно для декоративных элементов

### Минусы

-   нельзя управлять элементами SVG
-   сложно менять цвет

------------------------------------------------------------------------

## 4. SVG Sprite (лучший способ для иконок)

Создаётся один файл со всеми иконками.

``` html
<svg class="icon">
  <use href="images/sprite.svg#icon-home"></use>
</svg>
```

Это позволяет использовать одну и ту же иконку много раз без
дублирования кода.

------------------------------------------------------------------------

# Основные теги SVG

## svg

Корневой элемент.

``` html
<svg viewBox="0 0 24 24"></svg>
```

## path

Основной элемент, который рисует форму.

``` html
<path d="M10 10 H 90 V 90 H 10 Z" />
```

## rect

Прямоугольник.

``` html
<rect x="10" y="10" width="100" height="50" />
```

## circle

Круг.

``` html
<circle cx="50" cy="50" r="40" />
```

## g

Группа элементов.

``` html
<g>
  <path d="..." />
  <path d="..." />
</g>
```

## symbol

Используется в sprite-файле.

``` html
<symbol id="icon-home" viewBox="0 0 24 24">
  <path d="..." />
</symbol>
```

## use

Вставляет символ.

``` html
<svg class="icon">
  <use href="#icon-home"></use>
</svg>
```

------------------------------------------------------------------------

# Самые важные атрибуты SVG

## viewBox

    viewBox="0 0 24 24"

Формат:

    min-x min-y width height

Это внутренняя система координат SVG.

Если нет viewBox --- SVG плохо масштабируется.

------------------------------------------------------------------------

## width и height

Можно задать прямо:

``` html
<svg width="24" height="24">
```

Но лучше управлять размером через CSS.

------------------------------------------------------------------------

## fill

Цвет заливки.

``` html
<path fill="red" />
```

## stroke

Цвет обводки.

``` html
<path stroke="blue" />
```

------------------------------------------------------------------------

# Лучший способ менять цвет --- currentColor

### Внутри SVG

``` html
<path fill="currentColor" />
```

### В CSS

``` scss
.icon {
  color: #2563eb;
}
```

Теперь иконка наследует цвет текста.

Это самый удобный подход.

------------------------------------------------------------------------

# Базовая система иконок через SCSS

``` scss
.icon {
  width: 1em;
  height: 1em;
  display: inline-block;
  flex-shrink: 0;
  vertical-align: middle;
  color: currentColor;
}
```

### Размеры

``` scss
.icon--16 { width:16px; height:16px; }
.icon--20 { width:20px; height:20px; }
.icon--24 { width:24px; height:24px; }
```

``` xml
<svg xmlns="http://www.w3.org/2000/svg">
  <symbol id="icon-home" viewBox="0 0 24 24">
    <path fill="currentColor"
      d="M12 3 3 10v11h6v-7h6v7h6V10z"/>
  </symbol>
</svg>
```

# Использование иконки из sprite

``` html
<button class="button">
  <svg class="icon icon--20">
    <use href="images/sprite.svg#icon-home"></use>
  </svg>
  Home
</button>
```

# Доступность (Accessibility)

Если иконка декоративная:

``` html
<svg aria-hidden="true">
  <use href="#icon-home"></use>
</svg>
```

Если иконка несёт смысл:

``` html
<svg role="img" aria-label="Поиск">
  <use href="#icon-search"></use>
</svg>
```

# Частые ошибки при работе с SVG

### 1. Жёсткий цвет

``` html
fill="#000"
```

Тогда CSS не сможет перекрасить иконку.

------------------------------------------------------------------------

### 2. Использование img

``` html
<img src="icon.svg">
```

CSS не видит внутренние path.

------------------------------------------------------------------------

### 3. Нет viewBox

SVG плохо масштабируется.

------------------------------------------------------------------------

### 4. SVG из Figma содержит мусор

-   defs
-   mask
-   clipPath
-   transform

Иконку часто нужно почистить.

------------------------------------------------------------------------

### 5. Путают fill и stroke

Если иконка рисуется линиями, нужно менять stroke.

# Рекомендуемая архитектура иконок

    images/
      icons/
        home.svg
        user.svg
        search.svg
      sprite.svg

Оригинальные иконки хранятся отдельно, а sprite собирается из них.

# Практика, которую стоит использовать

Для большинства сайтов:

-   использовать **SVG sprite**
-   внутри ставить **currentColor**
-   обязательно оставлять **viewBox**
-   размер менять через CSS
-   цвет менять через color

------------------------------------------------------------------------

# Мини‑шпаргалка

## Изменить размер

``` scss
.icon {
  width: 24px;
  height: 24px;
}
```

## Изменить цвет

SVG:

``` html
<path fill="currentColor" />
```

CSS:

``` scss
.icon {
  color: red;
}
```

## Контурная иконка

``` html
<path stroke="currentColor" fill="none" />
```
