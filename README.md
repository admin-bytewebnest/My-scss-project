# My-scss-project

[Вопрос-1] `Что такое path?`

## path

Основной элемент, который рисует форму.

`Пример кода`

```html
<path d="M10 10 H 90 V 90 H 10 Z" />
```

[Вопрос-2] `Что такое rect?`

## rect

Прямоугольник.

`Пример кода`

```html
<rect x="10" y="10" width="100" height="50" />
```

[Вопрос-3] `Что такое circle?`

## circle

Круг.

```html
<circle cx="50" cy="50" r="40" />
```

[Вопрос-4] `Что такое g?`

## g

Группа элементов.

```html
<g>
  <path d="..." />
  <path d="..." />
</g>
```

[Вопрос-5] `Что такое symbol?`

## symbol

Используется в sprite-файле.

```html
<symbol id="icon-home" viewBox="0 0 24 24">
  <path d="..." />
</symbol>
```

viewBox нужно брать из каждой SVG-иконки и не делать его одинаковым для всех.

`Что будет, если ставить один viewBox всем подряд?`

иконка будет обрезана
или сожмётся
или будет с огромными отступами
или вообще "сломается визуально"

`viewBox` - это лист бумаги
`path` - это рисунок на листе
`width/height` - это насколько ты увеличиваешь этот лист на экране

[Алгоритм]

1. Берёшь SVG
2. Копируешь viewBox
3. Копируешь только содержимое (path, rect и т.д.)
4. Вставляешь в <symbol>

[Вопрос-6] `Что такое use?`

## use

Вставляет символ.

```html
<svg class="icon">
  <use href="#icon-home"></use>
</svg>
```

[Вопрос-7] `Какие есть самые важные атрибуты в SVG?`

1. ## viewBox
   Это внутренняя система координат SVG.

Если нет `viewBox` или не перенесли из существующей иконки в отдельный `sprite.svg` --- SVG плохо масштабируется.

2. ## width и height

Можно задать прямо в `html`:

```html
<svg width="24" height="24"></svg>
```

Но лучше управлять размером через CSS.

[ВАЖНО] - Удалить все размеры внутри `SVG`

3. ## fill
   Цвет заливки.

```html
<path fill="red" />
```

# Лучший способ менять цвет --- currentColor

### Внутри SVG

```html
<path fill="currentColor" />
```

### В CSS

```scss
.icon {
  color: #2563eb;
}
```

4. ## stroke
   Цвет обводки.

```html
<path stroke="blue" />
```

# Лучший способ менять цвет --- currentColor

### Внутри SVG

```html
<path stroke="currentColor" />
```

### В CSS

```scss
.icon {
  color: #2563eb;
}
```

Теперь иконка наследует цвет текста.
Это самый удобный подход.

`Когда использовали currentColor - мы можем`

```html
<button>
  <svg class="icon icon--md" aria-hidden="true">
    <use href="images/sprite.svg#svg-home"></use>
  </svg>
  Отправить
</button>
```

```scss
button {
  color: #2563eb;
}
```

`!!` - Когда используется `currentColor` — мы можем управлять цветом SVG через родителя.

`!!` В этом случае перекрасится не только текст кнопки, но и SVG-иконка,
потому что внутри SVG используется fill="currentColor",
а currentColor берёт значение из свойства color родителя.

`currentColor` = это значение CSS свойства color текущего элемента


`Что еще наследуется?`

```css
.icon {
  width: 1em;
  height: 1em;
}
```

`!!` - Наследуется размер который установлен для родительского элемента

# Частые ошибки при работе с SVG

### 1. Жёсткий цвет

``` html
fill="#000"
```

Тогда CSS не сможет перекрасить иконку.

------------------------------------------------------------------------

### 2. Использование img

``` html
<img src="icon.svg">
```

CSS не видит внутренние path.

------------------------------------------------------------------------

### 3. Нет viewBox

SVG плохо масштабируется.

------------------------------------------------------------------------

### 4. SVG из Figma содержит мусор

-   defs
-   mask
-   clipPath
-   transform

Иконку часто нужно почистить.

------------------------------------------------------------------------

### 5. Путают fill и stroke

Если иконка рисуется линиями, нужно менять stroke.

------------------------------------------------------------------------
